# Kaspi + WB Search

Простой MVP на Python.

Идея:

- вводишь один запрос
- сайт делает поиск в `Wildberries` и `Kaspi`
- ниже показывает результаты из двух источников

Без сложного интерфейса и без лишней архитектуры.

## Что где лежит

- `app/main.py` — FastAPI приложение и маршруты
- `app/services/marketplaces.py` — вся логика запросов в WB и Kaspi
- `app/schemas.py` — модели ответа
- `app/templates/index.html` — простая HTML страница
- `app/static/styles.css` — минимальные стили

## Как это работает

### Wildberries

Для WB идет один `curl` запрос в их публичный search endpoint.

Из ответа берем:

- название
- цену
- бренд
- продавца
- ссылку на товар

### Kaspi

У Kaspi сначала открывается обычная страница поиска, чтобы получить сессию.
Потом идет второй запрос в их JSON endpoint с результатами.

Из ответа берем:

- название
- цену
- бренд
- продавца
- ссылку на товар

## Ограничения

- это не официальное единое API двух маркетплейсов
- `Kaspi` завязан на их текущий frontend endpoint
- `Wildberries` может иногда отдавать `429`, если часто дергать поиск
- у `Wildberries` в этом MVP нет стабильной картинки товара, поэтому сейчас вывод идет без фото

## Запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

После запуска:

- сайт: `http://127.0.0.1:8000`
- API: `http://127.0.0.1:8000/api/search?q=iphone`

## Настройки

Можно переопределить:

```bash
set WB_DEST=-1257786
set KASPI_CITY_CODE=750000000
```
