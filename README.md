# Kaspi + WB Search

Простой Python MVP, который ищет товары сразу по двум маркетплейсам:

- `Wildberries`
- `Kaspi Магазин`

Сайт построен на `FastAPI` и отдает:

- HTML-страницу с поиском и карточками
- JSON API `GET /api/search?q=...`

## Что уже умеет

- один поисковый запрос на обе площадки
- вывод названия, цены, продавца, рейтинга и ссылки на товар
- отдельные блоки результатов по каждому маркетплейсу
- базовая обработка ошибок по источникам

## Ограничения MVP

- `Kaspi` работает через их внутренний frontend endpoint, поэтому поиск зависит от текущей структуры сайта
- для `Kaspi` цены и ссылки сейчас идут в дефолтном городе `750000000` (Алматы)
- `Wildberries` карточки загружаются без стабильного image URL, поэтому для них сейчас используется fallback-блок вместо фото
- это не официальный публичный агрегаторный API обеих площадок, а рабочий MVP на основе публичных веб-источников

## Запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

После запуска:

- сайт: `http://127.0.0.1:8000`
- API: `http://127.0.0.1:8000/api/search?q=iphone`

## Настройки окружения

Можно переопределить значения через переменные окружения:

```bash
set WB_DEST=-1257786
set KASPI_CITY_CODE=750000000
```
