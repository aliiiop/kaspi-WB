# Kaspi + WB Search

Идея:

вводишь один запрос
ниже показывает результаты из двух источников


Как это работает

Wildberries

Для WB идет один `curl` запрос в их публичный search endpoint.

Из ответа берем:

название
цену
бренд
продавца
ссылку на товар

Kaspi

У Kaspi сначала открывается обычная страница поиска, чтобы получить сессию.
Потом идет второй запрос в их JSON endpoint с результатами.

Из ответа берем:

название
цену
бренд
продавца
ссылку на товар

Ограничения

это не официальное единое API двух маркетплейсов
`Kaspi` завязан на их текущий frontend endpoint
`Wildberries` может иногда отдавать `429`, если часто дергать поиск
у `Wildberries` в этом MVP нет стабильной картинки товара, поэтому сейчас вывод идет без фото(и у каспи тоже не выводит)

Запуск


python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
